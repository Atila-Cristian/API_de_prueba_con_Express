

DELETE
Sirve para eliminar información del servidor.

PATCH
Sirve para actualizar parcialmente un recurso.
 Cambia solo algunos campos (no todo).

HEAD
Es como GET, pero sin devolver el cuerpo.
 Solo headers.

OPTIONS
Sirve para saber qué métodos están permitidos en una ruta.
 Muy usado con CORS.

TRACE
Sirve para diagnóstico (ver el recorrido del request).
 Casi no se usa.

CONNECT
Sirve para crear un túnel (por ejemplo HTTPS a través de un proxy).
 Muy específico.
